### The Revolution in HID Security Research

FLUCKY represents a paradigm shift in wireless Human Interface Device (HID) security testing tools. Built from the ground up for modern security professionals, academic researchers, and ethical penetration testers, FLUCKY transcends the limitations of traditional HID attack platforms by introducing wireless Bluetooth Low Energy (BLE) connectivity, psychological operation capabilities, and enterprise-grade operational security features.

#### Why FLUCKY Exists - A Story of Necessity

The landscape of cybersecurity has evolved dramatically over the past decade. Traditional HID testing tools, while revolutionary in their time, have begun to show their age. These legacy platforms suffered from several critical limitations that hindered effective security assessment and awareness training:

**Traditional Limitations:**
- **Physical Access Requirement**: Legacy tools required direct physical connection to target systems, creating obvious indicators of compromise and limiting realistic attack simulation scenarios
- **Limited Stealth Capabilities**: Visual indicators, logging outputs, and predictable execution patterns made detection trivial
- **Absence of Psychological Elements**: Purely technical attacks without the human element that characterizes real-world sophisticated threats
- **Static Payload Execution**: Predictable command sequences easily detected by modern endpoint protection solutions
- **Wireless Limitations**: Tethered operation reduced operational flexibility and realism in security assessments

FLUCKY addresses each of these limitations while introducing innovative features that reflect the sophisticated threat landscape of 2025 and beyond. The fundamental breakthrough of FLUCKY lies in its automatic BLE connectivity - the device appears as a standard Bluetooth keyboard without requiring manual pairing, enabling seamless integration into authorized security testing scenarios.

### Step-by-Step Installation Guide

#### Step 1: Environment Setup

**Prerequisites:**
- Python 3.7 or higher installed
- pip package manager
- USB drivers for ESP32 (CP210x or CH340 depending on board)

**Installing esptool:**

```bash
# Install esptool via pip
pip install esptool

# Verify installation
esptool.py --version
```

#### Step 2: Firmware Flashing

**Identifying the Correct COM Port:**

```bash
# Windows - Check Device Manager under Ports (COM & LPT)
# Linux - Check /dev/ttyUSB* or /dev/ttyACM*
# macOS - Check /dev/cu.usbserial* or /dev/cu.usbmodem*
```

**Flashing the Firmware:**

```bash
# Erase flash first (recommended for clean install)
esptool.py --port COM3 erase_flash

# Flash FLUCKY firmware (replace COM3 with your port)
esptool.py --port COM3 --baud 921600 write_flash 0x1000 esp32_s3_flucky_backup.bin

# Verify the flash (optional but recommended)
esptool.py --port COM3 verify_flash 0x1000 esp32_s3_flucky_backup.bin
```

**Platform-Specific Commands:**

```bash
# Windows
esptool.py --port COM3 --baud 921600 write_flash 0x1000 esp32_s3_flucky_backup.bin

# Linux (may require sudo)
sudo esptool.py --port /dev/ttyUSB0 --baud 921600 write_flash 0x1000 esp32_s3_flucky_backup.bin

# macOS
esptool.py --port /dev/cu.usbserial-0001 --baud 921600 write_flash 0x1000 esp32_s3_flucky_backup.bin
```

#### Step 3: Initial Configuration

**Power On Sequence:**
1. Connect ESP32 via USB or external power
2. LED should blink 3 times indicating successful boot
3. Open Serial Monitor at 115200 baud rate

### Encryption Implementation Details

#### XOR Cipher Implementation

FLUCKY uses XOR encryption with key cycling for script storage protection.

**Algorithm:**

```
For each byte in script:
  encrypted_byte = plaintext_byte XOR key_byte[key_position]
  key_position = (key_position + 1) MOD key_length
```

**Security Properties:**

| Property | Implementation |
|----------|---------------|
| Key cycling | Repeats key through entire script |
| Deterministic | Same key + script = same encrypted output |
| Reversible | Same operation encrypts and decrypts |
| Performance | Minimal CPU overhead on ESP32 |
